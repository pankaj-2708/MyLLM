# MiniFinLLM: 110M Parameter Financial Language Model from Scratch

An end-to-end 110M parameter Large Language Model built entirely from scratch, pre-trained on the EDGAR financial corpus, and aligned using Supervised Fine-Tuning (SFT) and Direct Preference Optimization (DPO). 

This project was built to master the complete LLM lifecycle—from designing the architecture and writing custom PyTorch dataloaders to navigating multi-GPU distributed training constraints.

[HuggingFace Model](https://huggingface.co/Pankaj121212/PM_MiniFinLLM) 

## 🚀 Key Engineering Highlights

* **Custom Architecture:** Implemented a GPT-2 style transformer natively in PyTorch, upgrading the original architecture by integrating Rotary Positional Embeddings (RoPE).
* **Memory-Optimized Data Pipeline:** Engineered a padding-free tokenization strategy using `np.memmap` to stream a 2.3B token dataset directly from binary files, drastically reducing RAM overhead.
* **Hardware-Constrained Training:** Successfully trained on 2x Kaggle T4 GPUs (16GB VRAM each) by calculating precise activation memory limits to maximize batch sizes without hitting OOM errors.
* **Distributed Data Parallel (DDP):** Authored a custom multi-GPU training loop using PyTorch DDP, implementing mixed-precision (FP16) and gradient accumulation to simulate optimal batch sizes.
* **End-to-End Alignment:** Managed the full post-training pipeline, standardizing diverse QA datasets into chat templates for SFT, and utilizing DPO for memory-efficient alignment over traditional PPO.

## 🛠️ Tech Stack

* **Deep Learning:** PyTorch, `torch.nn.parallel.DistributedDataParallel`
* **Model Ecosystem:** Hugging Face `transformers`, `tokenizers`, `trl` (SFTTrainer)

## 📉 Training Pipeline Overview

### 1. Pre-Training
* **Dataset:** EDGAR corpus (Business Overviews, Risk Factors, MD&A).
* **Scale:** ~2.36 Billion tokens.
* **Strategy:** Documents were chunked into fixed-length sequences (1024) with EOS tokens only at true document boundaries, eliminating the need for padding tokens. 
* **Optimization:** AdamW optimizer with decoupled weight decay, cosine learning rate scheduler with warmup, and gradient clipping.

### 2. Supervised Fine-Tuning (SFT)
* **Dataset:** 21K samples compiled from multiple financial QA datasets.
* **Strategy:** Standardized datasets into a unified conversational format. Used a custom chat template with generation tags (`{% generation %}`) to ensure the loss function only penalized assistant response tokens, not user prompts.

### 3. Direct Preference Optimization (DPO)
* **Dataset:** `argilla/ultrafeedback-binarized-preferences-cleaned`
* **Strategy:** Replaced hardware-intensive PPO (which requires 4 models in memory) with DPO (requiring only 2). This allowed for successful alignment training within the strict 16GB VRAM limits of the T4 GPUs.

## 💻 Quick Start & Usage

The model and tokenizer are fully integrated with the Hugging Face ecosystem. 

```python
import torch
from transformers import AutoModelForCausalLM, AutoTokenizer

device = "cuda" if torch.cuda.is_available() else "cpu"

# Load Model and Tokenizer
model = AutoModelForCausalLM.from_pretrained(
    "Pankaj121212/PM_MiniFinLLM", 
    trust_remote_code=True
).to(device)

tokenizer = AutoTokenizer.from_pretrained(
    "Pankaj121212/PM_MiniFinLLM", 
    trust_remote_code=True
)

# Inference
prompt = "What are the primary market risks associated with interest rate fluctuations?"
inputs = tokenizer(prompt, return_tensors="pt").to(device)
outputs = model.generate(**inputs, max_new_tokens=100)

print(tokenizer.decode(outputs[0], skip_special_tokens=True))
